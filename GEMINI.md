# Projeto

Sistema SaaS premium desenvolvido em:

- Next.js
- TypeScript
- TailwindCSS
- Vercel

# Objetivo

Criar uma experiência premium parecida com:

- Vercel
- Stripe
- Linear
- Notion

# Regras

Antes de alterar qualquer arquivo:

1. Analise toda a arquitetura.
2. Preserve funcionalidades.
3. Preserve SEO.
4. Preserve responsividade.
5. Preserve performance.

# Design

Sempre utilizar:

- Glassmorphism leve
- Bordas suaves
- Animações modernas
- Framer Motion
- Dark Mode elegante

# Deploy

Toda alteração deve funcionar na Vercel.

Sempre executar:

npm run build

e corrigir erros encontrados.

# Segurança

Nunca expor:

- .env
- tokens
- API keys

# Quando eu pedir melhorias

1. Analise o projeto inteiro.
2. Crie um plano.
3. Execute mudanças gradualmente.
4. Nunca quebre funcionalidades.


# Ambiente

Sistema operacional: Windows 10

Terminal padrão: PowerShell

Sempre gerar comandos compatíveis com PowerShell.

Evitar:
- &&
- ||
- bash específico
- comandos Linux exclusivos

Preferir:
- ;
- PowerShell nativo
- npm
- git compatível com Windows


Nunca executar:

git push
git merge
git rebase
git reset --hard

sem solicitar confirmação explícita.

Antes de qualquer push:

1. Executar npm run build
2. Corrigir erros
3. Mostrar resumo das alterações
4. Pedir autorização
